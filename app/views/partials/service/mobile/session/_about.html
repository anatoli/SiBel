Управление сессиями{{SecondPage_About_Title_1 |translate}}

При агрегации множества сессий Интернет-соединений, весьма продолжительных (несколько дней) и не имеющих признака окончания, средства агрегации данных, такие как коллекторы IUM, могут оказаться неэффективными из-за ограниченного размера памяти. {{SecondPage_About_Text_1_1 |translate}}

Использование в качестве инструмента агрегации БД также накладывает ограничения, например, скорость обработки, дороговизна, сложность и т. д.{{SecondPage_About_Text_1_1 |translate}}


Специализированный агрегатор с гибкими средствами настройки позволяет управлять данными с любыми ограничениями размера памяти и количества используемых ядер процессоров. {{SecondPage_About_Text_2_2 |translate}}

Гибкая конфигурация позволяет максимально задействовать имеющиеся ресурсы серверов для решения поставленной задачи и задавать весьма широкие параметры для формирования результатов. Таким образом для сессий GPRS и PTPT может быть использован 20-часовой таймаут для определения окончания сессии. {{SecondPage_About_Text_1_1 |translate}}
